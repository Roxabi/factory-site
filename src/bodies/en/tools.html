Roxabi Factory · Tools

Available Tools

The tool layer is where Factory agents reach into the world — voice, image, visual artifacts, video, curation, social publishing. Each tool is a capability surface backed by a provider; workers consume tools but are not tools. The CLIs listed here run standalone today. Wiring them into the Factory engine as first-class tool-plane providers is on the roadmap, tracked under #1670.

This page lists tool-plane instances. For the model — taxonomy, runtime vocabulary, domain-nature split — see Tool Architecture.

Roxabi primitives

Self-hostable capability CLIs

Open-source tools built and maintained by Roxabi. Each works standalone on your own hardware — no cloud lock-in, no telemetry. When the tool-plane lands, they will register as satellite providers on the Factory NATS bus.

voiceCLI

Live

Voice · TTS / STT

One CLI for local speech synthesis and transcription. Drives Qwen3-TTS, Chatterbox, and Whisper from a single Markdown script. A NATS satellite enables cross-host voice — the same audio pipeline from any node on the network.

voiceCLI on roxabi.dev →

imageCLI

On-demand

Image · generation

One CLI for local image generation. FLUX.2, FLUX.1 (+ PuLID face lock), and SD3.5 behind one command. Quantization and VRAM managed for you. A NATS satellite for cross-host rendering. Activated when a GPU node is online.

imageCLI on roxabi.dev →

roxabi-forge

Live

Visual artifacts · diagrams, galleries, slides

A Claude Code plugin that turns "explain this" into a self-contained HTML artifact — diagrams, galleries, decks, guides. Eight skills, one-command deploy. Candidate for a visual-artifacts tool domain in the Factory plane.

roxabi-forge on roxabi.dev →

roxabi-production

On-demand

Video · generation

Video generation pipeline. Combines image generation, audio synthesis, and scene composition into finished video output. Runs on-demand on GPU hardware. Candidate for a video tool domain.

Third-party publishing surface
